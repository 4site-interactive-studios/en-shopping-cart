## Development

- Clone the repository
- Run `npm install`
- Make changes to the `src` folder
- Run `npm run build` to compile the changes
- Deploy the content of the `dist` folder to Engaging Networks

*NB: On windows you may also need to install the node_env npm package:*

    npm install -g win-node-env

## Add Button mode (single CTA → quantity controls flip)

Cards can opt-in to a single-button "Add to Cart" experience instead of the default `- N +` quantity controls. The button shows a static Noto emoji on the left and a label on the right; on hover the emoji animates. Clicking flips the button (3D rotateY) to reveal the regular `-` / quantity / `+` controls, with a free-form label below.

### Quick start

Drop a `.sc-add` element anywhere inside the card. The cart code reads its attributes, removes the original element, and renders the flip widget in the standard quantity slot.

```html
<button class="sc-add"
        data-icon="panda"
        data-quantity-label="in care package">
  Add to Care Package
</button>
```

Two-line button (small subtitle + main title):

```html
<button class="sc-add"
        data-icon="evergreen-tree"
        data-subtitle="Join 50k+ donors"
        data-quantity-label="trees planted">
  Plant Trees Today
</button>
```

Subtitle below the title and custom font sizes:

```html
<button class="sc-add"
        data-icon="wrapped-gift"
        data-subtitle="100% to the cause"
        data-subtitle-position="below"
        data-quantity-label="gifts given"
        data-title-size="20px"
        data-subtitle-size="13px">
  Give a Gift
</button>
```

### Attributes

| Attribute | Required | What it does |
|---|---|---|
| `class="sc-add"` | yes | Marker that triggers the Add-button widget for this card. |
| `data-icon="<name>"` | recommended | Animated emoji name (see the gallery below). Omit to render the button without an emoji. |
| `data-quantity-label="..."` | recommended | Small label shown under the quantity number after the user adds the item (e.g., "in care package"). |
| `data-subtitle="..."` | optional | Smaller line above (or below) the main title. Hidden on mobile. |
| `data-subtitle-position="above" \| "below"` | optional | Position of the subtitle. Defaults to `above`. |
| `data-title-size="20px"` | optional | Override the main-line font size (any CSS length). |
| `data-subtitle-size="13px"` | optional | Override the subtitle font size. |
| Button inner text | yes | Becomes the main bottom line of the button. |

### CSS variables

Override these on `.sc-cards-quantity.sc-cards-with-add` (per card or globally) to tweak the look. Colors are normally driven automatically by the card theme (`bg-teal`, `bg-orange`, etc.):

```css
#en-shopping-cart .sc-cards-with-add {
  --sc-add-height: 55px;          /* button + back-side height */
  --sc-add-radius: 10px;
  --sc-add-title-size: 18px;
  --sc-add-subtitle-size: 12px;
  --sc-add-emoji-size: 32px;
  /* Theme-driven defaults below — change per theme via the theme mixin: */
  --sc-add-bg: #ffffff;
  --sc-add-text: #3678ba;
  --sc-add-border: #3678ba;
  --sc-add-back-bg: #ffffff;
  --sc-add-back-border: #3678ba;
  --sc-add-back-text: #3678ba;
}
```

### Behavior

- **Idle**: static emoji + button text on a white background with a theme-colored border.
- **Hover**: the emoji animates (Noto Lottie/WebP loaded on demand) and the button lifts (stronger shadow + 1px translate). The background stays white so the animation is the focal point.
- **Click**: the quantity goes 0 → 1 and the card flips 3D (0.6s rotateY) to reveal the `-` / quantity / `+` controls with the data-quantity-label centered underneath.
- **Quantity ≥ 1**: stays on the back face; `-` and `+` adjust the count.
- **Quantity → 0**: flips back to the Add button.
- **State**: persists per-visitor via `localStorage`, keyed on the EN `campaignPageId` (same as the default quantity cards).
- **Mobile (≤ 690px)**: button height shrinks to 46px, emoji to 24px, title to 13px with up to two wrapped lines, subtitle is hidden, +/- icons shrink to 22px, and the card's bottom padding tightens to 65px.

### Theme integration

The Add button picks up its colors automatically from the card's `bg-*` theme. All themes use a white background with theme-colored text and border so the animated emoji stays readable on every card. On hover the button lifts via shadow only — the background never darkens, so the emoji animation remains visible.
